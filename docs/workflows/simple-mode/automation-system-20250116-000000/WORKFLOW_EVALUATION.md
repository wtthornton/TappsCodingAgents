### üîß **Suggestion 2: Hybrid Execution Model**

**Problem**: All-or-nothing approach (either fully manual or fully automatic)

**Solution**: Support both manual and automatic execution

**Design**:
- **Mode 1: Automatic**: Framework executes agents, generates content
- **Mode 2: Manual**: User provides content, framework structures it
- **Mode 3: Hybrid**: Framework executes some steps, user provides others
- Configuration flag: `automation_level: "full" | "manual" | "hybrid"`

**Benefits**:
- Flexibility for different use cases
- User control when needed
- Automation when desired
- Progressive adoption

**Implementation Approach**:
- Add execution mode to BuildOrchestrator
- Support content injection for manual steps
- Skip agent execution when content provided
- Merge manual and automatic content
