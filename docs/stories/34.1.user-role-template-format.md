# Story 34.1: User Role Template Format & Structure

<!-- Source: implementation/EPIC_06_User_Role_Templates.md -->

## Status

**Ready for Review**

## Story

**As a** framework user,
**I want** a standardized format for user role templates,
**so that** I can customize agent behavior based on my role (developer, PM, QA, etc.).

## Acceptance Criteria

1. User role template format (YAML) is defined with sections: verbosity, workflow_defaults, expert_priorities, documentation_preferences, review_depth
2. Template structure includes role metadata (role_id, description, use_cases)
3. Template structure is created with examples
4. Format is documented with explanations
