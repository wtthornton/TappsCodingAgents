Capabilities & Limitations

Understanding what TappsCodingAgents can and cannot do helps you use it effectively. This guide sets realistic expectations and helps you choose the right approach for your tasks.

What We Can Do

‚úÖ Code Generation & Implementation

  • Generate complete features - Full implementations with models, routes, schemas, tests
  • Multi-file coordination - Create/modify multiple related files consistently
  • Follow patterns - Maintain architectural patterns and project structure
  • Context-aware coding - Understand existing codebase and integrate properly
  • Generate tests - Unit tests, integration tests, API tests

‚úÖ Code Quality & Review

  • Code review - Comprehensive reviews with scoring
  • Quality metrics - Complexity, security, maintainability, performance scores
  • Security scanning - Vulnerability detection, dependency checks
  • Linting & type checking - Static analysis and type validation
  • Code improvement - Refactoring suggestions and implementations

‚úÖ Planning & Design

  • Requirements analysis - Extract and structure requirements
  • User stories - Generate stories with acceptance criteria
  • Architecture design - System architecture and component design
  • API design - RESTful API specifications and contracts
  • Data modeling - Database schemas and data structures

‚úÖ Documentation

  • API documentation - OpenAPI/Swagger specs, endpoint docs
  • Code documentation - Docstrings, comments, README files
  • Architecture docs - System design documentation
  • User guides - Usage documentation and tutorials

‚úÖ Workflow Orchestration

  • Multi-agent coordination - Orchestrate multiple agents in sequence
  • Quality gates - Automatic quality checks and thresholds
  • Artifact tracking - Track files and documents created at each stage
  • Conditional logic - Branch workflows based on conditions
  • Error handling - Retry logic and failure recovery

‚úÖ Integration & Configuration

  • Cursor integration - Skills and Background Agents
  • Project profiling - Automatic project characteristic detection
  • Expert configuration - Domain-specific expert guidance
  • Context management - Tiered context for efficient token usage
  • Hardware optimization - Profile-based performance tuning

What We Cannot Do

‚ùå Real-Time Execution

  • Cannot execute code - We generate code but don't run it
  • Cannot test runtime behavior - No execution or debugging of running applications
  • Cannot interact with live systems - No database connections, API calls, or system interactions

‚ùå Large-Scale Refactoring

  • Limited to workflow scope - Refactoring is scoped to the files/workflow context
  • No automatic codebase-wide changes - Won't automatically refactor entire large codebases
  • Requires explicit instructions - Need to specify what to refactor

‚ùå Complex Business Logic

  • May miss edge cases - Complex business rules may need human review
  • Domain expertise required - Complex domain logic benefits from expert configuration
  • May need iteration - Complex logic may require multiple passes

‚ùå Design & UI Creation

  • No visual design - Cannot create UI mockups or visual designs
  • Code-only UI - Can generate React/Vue components but not design them visually
  • No image generation - Cannot create images, icons, or graphics

‚ùå External Dependencies

  • Cannot install packages - Won't run pip install, npm install, etc.
  • Cannot configure services - No AWS, Docker, or infrastructure configuration
  • Cannot access external APIs - No calls to third-party services

‚ùå Git Operations

  • No automatic commits - Won't commit code to git
  • No branch management - Won't create or merge branches
  • Read-only git - Can read git history for context but won't modify it

‚ùå Deployment

  • No deployment automation - Won't deploy to servers or cloud
  • No CI/CD configuration - Can generate config files but won't set up pipelines
  • No infrastructure provisioning - Won't create cloud resources

When to Use Workflows

Use workflows (the primary approach) for:

  • New features - Complete feature implementation
  • Multi-file changes - Changes affecting multiple files
  • Quality-focused work - When you need quality gates and reviews
  • Complete implementations - End-to-end feature development
  • Architectural work - System design and planning
  • Production-ready code - When you need tests, security scans, documentation
Rule of Thumb: Use workflows for 90% of your development work. They provide better coordination, quality, and consistency than individual agent calls.

When to Use Individual Agents

Use individual agents for:

  • Quick reviews - Single-file code review
  • Simple refactoring - Refactoring within a single file
  • Quick fixes - Small bug fixes in one file
  • Analysis - Understanding code or requirements
  • Scoring - Quick quality score calculation
  • Prompt enhancement - Enhancing prompts before workflows
Individual Agent Examples 
# Quick review
tapps-agents reviewer review src/utils/helpers.py

# Quick score
tapps-agents reviewer score src/api/routes.py

# Simple refactor
tapps-agents implementer refactor src/models.py "Extract validation logic"

# Prompt enhancement
tapps-agents enhancer enhance "Add user authentication"

Limitations & Considerations

Token Limits

  • Context size - Very large codebases may exceed context limits
  • Tiered context helps - Uses appropriate tiers to stay within limits
  • May need chunking - Very large files may need to be processed in parts

Model Quality

  • Depends on Cursor's configured model - Quality varies by the model configured in Cursor
  • May need iteration - First attempt may not be perfect
  • Review recommended - Always review generated code

Complexity

  • Very complex features - May need to be broken into smaller workflows
  • Domain-specific logic - Benefits from expert configuration
  • Legacy code - May struggle with very old or unusual code patterns

Accuracy

  • Not 100% accurate - Generated code may have bugs or need refinement
  • Requires testing - Always test generated code
  • Human review needed - Especially for critical or security-sensitive code

Best Practices for Working Within Limitations

1. Break Down Complex Tasks

If a feature is very complex, break it into smaller workflows:

Breaking Down Tasks 
# Instead of one massive workflow:
# tapps-agents workflow full "Build entire e-commerce platform"

# Break it down:
tapps-agents workflow full "Build user authentication and authorization"
tapps-agents workflow full "Build product catalog and search"
tapps-agents workflow full "Build shopping cart and checkout"
tapps-agents workflow full "Build order management"

2. Use Iterative Approach

Don't expect perfection on the first try. Use workflows iteratively:

  1. Run workflow for initial implementation
  2. Review and test the code
  3. Run "fix" or "quality" workflow for improvements
  4. Iterate until satisfied

3. Configure Experts for Domain Work

For domain-specific projects, configure experts to improve quality:

Expert Configuration 
tapps-agents setup-experts add
# Configure finance, healthcare, or other domain experts

4. Review Generated Code

Always review code, especially for:

  • Security-sensitive operations
  • Business-critical logic
  • Performance-sensitive code
  • Integration points

5. Test Thoroughly

Workflows generate tests, but:

  • Review test coverage
  • Add edge case tests if needed
  • Run tests and fix any failures
  • Add integration tests for complex features

Troubleshooting Common Issues

Issue: Workflow Fails Quality Gates

Solution: Review the feedback, fix issues, and run again. Or use the "quality" workflow to improve code.

Issue: Generated Code Doesn't Match Intent

Solution: Be more specific in your prompt. Include constraints, examples, or reference existing patterns.

Issue: Context Too Large

Solution: The system automatically uses tiered context. If issues persist, break the task into smaller workflows.

Issue: Missing Dependencies

Solution: Workflows generate code but don't install packages. Manually install dependencies after code generation.

Issue: Integration Issues

Solution: Review integration points manually. Workflows may not catch all integration nuances.
