Overview

TappsCodingAgents is a specification framework for defining, configuring, and orchestrating coding agents. It provides a standardized approach to AI-powered software development with a two-layer agent model.

What is TappsCodingAgents?

TappsCodingAgents provides:

  • 13 Workflow Agents: Specialized agents covering the complete software development lifecycle (Planning, Design, Development, Testing, Quality, Operations, Orchestration, Enhancement)
  • 16 Built-in Experts: Framework-controlled technical domain experts (Security, Performance, Testing, Data Privacy, Accessibility, UX, Code Quality, Software Architecture, DevOps, Documentation, AI Frameworks, Observability, API Design, Cloud Infrastructure, Database, Agent Learning)
  • Industry Experts: Project-defined business domain knowledge with weighted decision-making
  • Simple Mode: Natural language interface for new users with automatic skill orchestration
  • Cursor AI Integration: Complete integration with Cursor IDE (all 7 phases complete)
  • YAML Workflow Definitions: Declarative, version-controlled orchestration

Architecture

TappsCodingAgents uses a two-layer agent model:

Layer Type Purpose Count
Knowledge Industry Experts Business domain authority N (per project)
Execution Workflow Agents SDLC task execution 13 (fixed)

Getting Started

Follow these steps to get started with TappsCodingAgents:

Recommended Flow: Installation ‚Üí Initialize ‚Üí Enable Simple Mode ‚Üí Start Coding

Step 1: Installation

Prerequisites:

  • Python 3.13+ (recommended: latest stable Python)
  • Git (to clone the repository)
  • Cursor IDE (for LLM operations via Skills)

Install from Source (Development):

Installation 
# Create virtual environment
python -m venv .venv

# Activate virtual environment
# Windows PowerShell:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

# Upgrade pip
python -m pip install --upgrade pip

# Install in editable mode
pip install -e .

Step 2: Initialize Project

Initialize your project to set up configuration, Cursor Rules, workflow presets, and Skills:

Initialize Project 
tapps-agents init

This creates:

  • .tapps-agents/config.yaml - Project configuration
  • .cursor/rules/ - Cursor Rules for AI assistance
  • workflows/presets/ - Workflow preset YAML files
  • .claude/skills/ - Cursor Skills
  • .cursor/background-agents.yaml - Background Agents config

Step 3: Enable Simple Mode (Recommended for New Users)

Simple Mode provides a natural language interface that automatically orchestrates multiple skills:

Enable Simple Mode 
# Enable Simple Mode
tapps-agents simple-mode on

# Or run the interactive onboarding wizard
tapps-agents simple-mode init

# Check status
tapps-agents simple-mode status

Step 4: Verify Setup

Verify that everything is set up correctly:

Verify CLI 
# Check CLI is available
tapps-agents --help

# Check version
tapps-agents --version

# Try a skill in Cursor
# Open Cursor chat and type:
@reviewer *help
@implementer *help
@tester *help
You're ready! You can now use TappsCodingAgents. Try Simple Mode in Cursor chat or use individual agent skills directly.

For detailed getting started instructions, see Getting Started Guide.

Simple Mode

Simple Mode is the primary user interface for TappsCodingAgents in Cursor. It provides natural language commands that automatically orchestrate multiple specialized skills through a structured workflow.

What is Simple Mode?

Simple Mode hides complexity while showcasing power. Instead of learning multiple agent commands, you use natural language and Simple Mode automatically:

  • Parses your intent
  • Routes to the appropriate orchestrator
  • Coordinates multiple agents in sequence
  • Reports progress with status updates
  • Summarizes results when complete

How to Use Simple Mode

Invoke with @simple-mode in Cursor chat:

Simple Mode Commands 
@simple-mode *build "Create a user authentication feature"
@simple-mode *review src/api/auth.py
@simple-mode *fix src/buggy.py "Fix the error"
@simple-mode *test src/api/auth.py
@simple-mode *full "Build a complete REST API"

Or use natural language:

Natural Language 
@simple-mode Build a user authentication feature
@simple-mode Review my authentication code
@simple-mode Fix the error in auth.py
@simple-mode Add tests for service.py

Simple Mode Commands

Command Description Skills Invoked
*build {description} Build new features @enhancer ‚Üí @planner ‚Üí @architect ‚Üí @designer ‚Üí @implementer ‚Üí @reviewer ‚Üí @tester
*review {file} Review code quality @reviewer ‚Üí @improver
*fix {file} {description} Fix bugs/errors @debugger ‚Üí @implementer ‚Üí @tester
*test {file} Generate tests @tester
*full {description} Full SDLC workflow All skills in sequence
*help Show help -
*status Check status -

Build Workflow (7 Steps)

When you use *build, Simple Mode executes ALL 7 steps in sequence:

  1. @enhancer *enhance - Enhanced prompt with requirements analysis, architecture guidance, quality standards
  2. @planner *plan - User stories with acceptance criteria, story points, estimates
  3. @architect *design - System architecture, component design, data flow
  4. @designer *design-api - Component specifications, API design, data models
  5. @implementer *implement - Code implementation following all specifications
  6. @reviewer *review - Quality scores (5 metrics), issues found, recommendations
  7. @tester *test - Test plan, browser compatibility, accessibility checklist, performance validation
‚ö†Ô∏è CRITICAL: When using *build commands, ALL 7 steps are executed and documentation artifacts are created. Do not skip steps.

Intent Detection

Simple Mode detects intent from keywords:

  • Build: build, create, make, generate, add, implement, develop, write, new, feature
  • Review: review, check, analyze, inspect, examine, score, quality, audit
  • Fix: fix, repair, resolve, debug, error, bug, issue, problem, broken
  • Test: test, verify, validate, coverage, testing
  • Full: full, complete, sdlc, lifecycle, everything

Configuration

Simple Mode can be configured in .tapps-agents/config.yaml:

Configuration 
simple_mode:
  enabled: true
  auto_detect: true
  show_advanced: false
  natural_language: true

Why Use Simple Mode?

  • Natural Language - Use plain English instead of learning multiple skill commands
  • Automatic Orchestration - Skills are coordinated automatically in the right sequence
  • Quality Gates - Automatic quality checks with loopback on failures
  • Progressive Disclosure - Start simple, unlock advanced features as you learn
  • Best Practices - Follows TappsCodingAgents workflow patterns automatically
  • Better Outcomes - Produces higher quality code with comprehensive documentation

For complete Simple Mode documentation, see Simple Mode Guide.

Features & Capabilities

Workflow Agents (13)

Complete SDLC coverage with specialized agents:

  • Planning: analyst, planner
  • Design: architect, designer
  • Development: implementer, debugger, documenter
  • Testing: tester
  • Quality: reviewer, improver
  • Operations: ops
  • Orchestration: orchestrator
  • Enhancement: enhancer

Expert System

16 Built-in Experts: Framework-controlled technical domain experts with 83 knowledge files:

  • Security, Performance, Testing, Data Privacy, Accessibility, UX
  • Code Quality, Software Architecture, DevOps, Documentation
  • AI Frameworks, Observability, API Design, Cloud Infrastructure, Database, Agent Learning

Industry Experts: Project-defined business domain experts with weighted decision-making and RAG integration.

Code Scoring System

Objective quality metrics with configurable thresholds:

  1. Complexity Score (0-10): Cyclomatic complexity analysis using Radon
  2. Security Score (0-10): Vulnerability detection using Bandit + heuristics
  3. Maintainability Score (0-10): Maintainability Index using Radon MI
  4. Test Coverage Score (0-10): Coverage data parsing + heuristic analysis
  5. Performance Score (0-10): Static analysis (function size, nesting depth, pattern detection)

YAML Workflow Definitions

Declarative, version-controlled orchestration with:

  • Strict Schema Enforcement: All YAML structures validated and executed
  • Task Manifest Generation: Auto-generated task checklists from workflow YAML
  • Automated Documentation: Cursor Rules auto-generated from workflow YAML
  • Background Agent Auto-Generation: Background Agent configs auto-generated from workflow steps
  • Dependency-Based Parallelism: Automatic parallel execution based on step dependencies

Cursor AI Integration (All 7 Phases Complete)

  • 13 Cursor Skills: All agents available as Cursor Skills
  • Simple Mode: Streamlined interface for new users
  • Custom Skills Support: Create, validate, and integrate custom Skills
  • Background Agents: Offload heavy tasks to cloud/remote
  • Multi-Agent Orchestration: Parallel execution with structured concurrency
  • Context7 Integration: KB-first caching + analytics
  • NUC Optimization: Resource monitoring, fallback strategy

Simple Mode Not Working?
  • Check status: tapps-agents simple-mode status
  • Enable if needed: tapps-agents simple-mode on
  • Verify configuration exists: cat .tapps-agents/config.yaml
Commands Not Recognized?
  • Use explicit keywords (build, review, fix, test)
  • Try using *build syntax instead of natural language
  • Use *show-advanced to access agent-specific commands
