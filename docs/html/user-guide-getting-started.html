Getting Started

Follow these steps in order to set up TappsCodingAgents for your project. The setup process is designed to be logical and progressive.

Recommended Flow: Initialize ‚Üí Profile ‚Üí Configure ‚Üí Setup Experts ‚Üí Start Coding

Step 1: Installation

Prerequisites

  • Python 3.13+ (recommended: latest stable Python)
  • Git (to clone the repository)
  • Ollama (optional, for local LLM execution)
Project-Scoped Installation: All installation steps use a virtual environment (venv) to ensure dependencies are installed only for your project scope, not globally.

Get the Source Code

First, clone or download the TappsCodingAgents repository:

Clone Repository 
# Clone the repository
git clone https://github.com/wtthornton/TappsCodingAgents.git
cd TappsCodingAgents

# Or download and extract the ZIP file from GitHub releases

Install TappsCodingAgents

Choose the installation method that best fits your needs:

Option 1: Install from GitHub Release (Recommended for End Users)

If you just want to use TappsCodingAgents without modifying the code, install from a GitHub release:

Install from GitHub Release 
# Create virtual environment (project-scoped, not global)
python -m venv .venv

# Activate virtual environment
# Windows PowerShell:
.venv\Scripts\activate
# Windows CMD:
.venv\Scripts\activate.bat
# macOS/Linux:
source .venv/bin/activate

# Upgrade pip to latest version (2025 best practice)
python -m pip install --upgrade pip

# Install from GitHub release (replace 2.0.6 with latest version)
# Download the .whl file from: https://github.com/wtthornton/TappsCodingAgents/releases
pip install https://github.com/wtthornton/TappsCodingAgents/releases/download/v2.0.6/tapps_agents-2.0.6-py3-none-any.whl

Option 2: Install from Source (Recommended for Development)

If you want to modify the code or contribute, install from source in editable mode:

Installation (Editable Mode - Development) 
# Create virtual environment (project-scoped, not global)
python -m venv .venv

# Activate virtual environment
# Windows PowerShell:
.venv\Scripts\activate
# Windows CMD:
.venv\Scripts\activate.bat
# macOS/Linux:
source .venv/bin/activate

# Upgrade pip to latest version (2025 best practice)
python -m pip install --upgrade pip

# Install TappsCodingAgents in editable mode (recommended for development)
# This allows you to modify the code and see changes immediately
pip install -e .

# Alternative: Regular installation (if you don't need to modify the code)
# pip install .
Why Editable Mode (-e)? Editable mode installs the package in "development mode" so changes to the source code are immediately available without reinstalling. This is the 2025 standard for local development.

What Gets Installed: The installation process will automatically install all required dependencies from pyproject.toml, including:

  • Core framework dependencies (pydantic, httpx, pyyaml, aiohttp, psutil)
  • Code analysis tools (radon, bandit, coverage)
  • Reporting tools (jinja2, plotly)
  • The tapps-agents CLI command

Verify Installation

Verify CLI 
# Verify the CLI command is available
tapps-agents --help

# Check version
tapps-agents --version

# Run the doctor command to verify environment
tapps-agents doctor

Troubleshooting Installation

Common Issues:
  • "Command not found" or "CommandNotFoundException" - Ensure your virtual environment is activated. Check that your prompt shows (.venv). On Windows PowerShell, use .venv\Scripts\activate.
  • "Could not find a version that satisfies the requirement tapps-agents" - This package is not published to PyPI. Install from GitHub releases using pip install https://github.com/wtthornton/TappsCodingAgents/releases/download/v2.0.6/tapps_agents-2.0.6-py3-none-any.whl or from source using pip install -e . from within the TappsCodingAgents directory.
  • "No module named 'tapps_agents'" - Make sure you're in the TappsCodingAgents directory and have run pip install -e . successfully.
  • "Python version not supported" - TappsCodingAgents requires Python 3.13+. Check your version: python --version and upgrade if needed.
  • Dependency conflicts - Ensure you're using a fresh virtual environment. Delete .venv and recreate: python -m venv .venv
Complete Installation Verification 
# 1. Verify Python version (must be 3.13+)
python --version

# 2. Verify virtual environment is active
# Your prompt should show (.venv)
# Windows PowerShell:
$env:VIRTUAL_ENV
# Linux/macOS:
echo $VIRTUAL_ENV

# 3. Verify installation
tapps-agents --version
tapps-agents doctor

# 4. Check installed packages
pip list | findstr tapps-agents  # Windows
pip list | grep tapps-agents     # Linux/macOS

Initialize Project

Step 3: Project Profiling

Project profiling happens automatically when you initialize your project. The framework detects your project characteristics to provide context-aware expert guidance.

Automatic Detection

The profiler analyzes your project and detects:

  • Deployment Type: local, cloud, enterprise
  • Tenancy Model: single-tenant, multi-tenant
  • User Scale: single-user, small-team, department, enterprise
  • Compliance Requirements: GDPR, HIPAA, PCI, SOC2
  • Security Level: basic, standard, high, critical
  • Technology Stack: frameworks, libraries, databases

View Your Project Profile

View Profile 
# View profile file
# Windows PowerShell:
Get-Content .tapps-agents\project-profile.yaml

# macOS/Linux:
cat .tapps-agents/project-profile.yaml

Example Profile

Example Project Profile 
project:
  name: "E-Commerce Platform"
  type: "web-application"

  type: "cloud"
  platform: "aws"
  tenancy: "multi-tenant"

  user_scale: "enterprise"
  expected_users: "10000+"

  - "GDPR"
  - "PCI-DSS"

  level: "high"
  requirements:
    - "encryption-at-rest"
    - "encryption-in-transit"
    - "audit-logging"

  databases: ["PostgreSQL", "Redis"]
Why Profiling Matters: The profile influences which experts are consulted, what security checks are performed, and which quality gates are applied. It ensures agents make contextually appropriate decisions.

Step 4: Configuration

Configure TappsCodingAgents to match your environment and preferences. The main configuration file is .tapps-agents/config.yaml.

Basic Configuration

Basic Config Example 
project:
  name: "My Project"
  root: "."

# Model Abstraction Layer (MAL)
mal:
  enabled: true
  ollama_url: "http://localhost:11434"
  default_model: "qwen2.5-coder:7b"
  fallback_provider: "anthropic"
  timeout: 60

# Code Scoring
scoring:
  enabled: true
  weights:
    complexity: 0.20
    security: 0.30
    maintainability: 0.25
    test_coverage: 0.15
    performance: 0.10
  thresholds:
    overall_score: 70

# Context Management
context:
  tiered_enabled: true
  default_tier: "tier1"
  cache_ttl: 300

Hardware Profile

Set your hardware profile for optimal performance, especially important for NUC or low-power systems.

Hardware Profile 
# Check current profile
tapps-agents hardware-profile

# Set profile (auto, nuc, development, workstation, server)
tapps-agents hardware-profile --set nuc

# Available profiles:
# - auto: Auto-detect (default)
# - nuc: Intel NUC / low-power systems
# - development: Development machines
# - workstation: High-performance workstations
# - server: Server environments

Verify Configuration

Doctor Command 
# Check environment and configuration
tapps-agents doctor

# JSON output
tapps-agents doctor --format json

Configure Business Domain Experts

For projects with specific domain requirements (finance, healthcare, etc.), configure business domain experts:

Setup Experts Wizard 
# Interactive wizard
tapps-agents setup-experts init

# Add expert manually
tapps-agents setup-experts add

# List configured experts
tapps-agents setup-experts list

Expert Configuration File

Experts are configured in .tapps-agents/experts.yaml:

Experts Configuration 
experts:
  - id: "finance-expert"
    name: "Finance Domain Expert"
    domain: "finance"
    knowledge_base: "experts/knowledge/finance/"
    weight: 0.51  # Primary expert (must sum to 1.0)
    
  - id: "healthcare-expert"
    name: "Healthcare Domain Expert"
    domain: "healthcare"
    knowledge_base: "experts/knowledge/healthcare/"
    weight: 0.24  # Secondary expert
    
  - id: "compliance-expert"
    name: "Compliance Expert"
    domain: "compliance"
    knowledge_base: "experts/knowledge/compliance/"
    weight: 0.25  # Tertiary expert
