User Guide

Complete guide to using TappsCodingAgents for AI-powered software development

Getting Started

Installation

Install TappsCodingAgents using pip:

Installation 
# Create virtual environment (recommended)
python -m venv .venv

# Activate virtual environment
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

# Install TappsCodingAgents
pip install tapps-agents

Quick Start: New Project

If you're starting a new project from scratch:

Step 1: Initialize Your Project

Run the initializer to set up your project with Cursor Rules, workflow presets, and configuration:

Initialize Project 
tapps-agents init

This creates:

  • .tapps-agents/config.yaml - Project configuration
  • .claude/skills/ - Cursor Skills for all 13 agents
  • workflows/presets/ - Pre-configured workflow definitions
  • .cursor/background-agents.yaml - Background agent configuration

Step 2: Verify Installation

Check your environment and installation:

Doctor Command 
# Check environment
tapps-agents doctor

# JSON output
tapps-agents doctor --format json

Step 3: Start Using

You're ready! See the Using Agents section below to get started.

Quick Start: Existing Project

If you're adding TappsCodingAgents to an existing project:

Step 1: Check if Already Initialized (Optional)

Check if your project already has TappsCodingAgents setup:

Check Configuration 
# Check for existing configuration
# Windows PowerShell:
Test-Path .tapps-agents\config.yaml

# macOS/Linux:
ls .tapps-agents/config.yaml 2>/dev/null || echo "Not initialized"

Step 2: Initialize (if needed)

If not initialized, or to add missing components, run the initializer:

Initialize Project 
tapps-agents init

Note: The init command is safe to run multiple times - it won't overwrite existing files. It will only add missing components.

Step 3: Verify Configuration

Check your environment and review your existing configuration:

Verify Setup 
# Check environment
tapps-agents doctor

# Review current config (optional)
# Windows PowerShell:
Get-Content .tapps-agents\config.yaml

# macOS/Linux:
cat .tapps-agents/config.yaml

Step 4: Start Using

You're ready! See the Using Agents section below to get started.

Using Agents

Reviewer Agent

The Reviewer Agent analyzes code quality, calculates scores, and provides feedback.

Review a File

Review Command 
# Review a Python file
tapps-agents reviewer review src/main.py

# Review with specific model
tapps-agents reviewer review src/main.py --model qwen2.5-coder:7b

# Output as JSON
tapps-agents reviewer review src/main.py --format json

Score Code Quality

Score Command 
# Calculate quality scores
tapps-agents reviewer score src/main.py

# Output includes:
# - Complexity Score (0-10)
# - Security Score (0-10)
# - Maintainability Score (0-10)
# - Test Coverage Score (0-10)
# - Performance Score (0-10)
# - Overall Score (0-100)

Implementer Agent

The Implementer Agent generates code, implements features, and performs refactoring.

Implement Command 
# Implement a feature
tapps-agents implementer implement "Add user authentication endpoint"

# Implement with file context
tapps-agents implementer implement "Add validation" --file src/api.py

Planner Agent

The Planner Agent creates development plans and generates user stories.

Planning Commands 
# Create a plan
tapps-agents planner plan "Add payment processing"

# Generate a user story
tapps-agents planner create-story \
    "User can reset password" \
    --epic "Authentication" \
    --priority high

Tester Agent

The Tester Agent generates and executes tests.

Testing Commands 
# Generate tests for a file
tapps-agents tester generate src/main.py

# Run tests
tapps-agents tester run tests/test_main.py

Enhancer Agent

The Enhancer Agent transforms simple prompts into comprehensive, context-aware prompts using a 7-stage pipeline.

Enhance Prompt 
# Full enhancement (all 7 stages)
tapps-agents enhancer enhance "Create a login system"

# Quick enhancement (stages 1-3)
tapps-agents enhancer enhance-quick "Add user authentication"

# Output formats
tapps-agents enhancer enhance "..." --format json
tapps-agents enhancer enhance "..." --format yaml

The 7-stage pipeline includes:

  1. Analysis - Detect intent, scope, domains
  2. Requirements - Gather functional/non-functional requirements
  3. Architecture - System design guidance
  4. Codebase Context - Inject relevant code context
  5. Quality Standards - Define security and testing thresholds
  6. Implementation Strategy - Task breakdown and order
  7. Synthesis - Combine into enhanced prompt

Workflows

What are Workflows?

Workflows are YAML-defined multi-step processes that orchestrate multiple agents to complete complex tasks. They support conditional steps, quality gates, and artifact tracking.

Running Workflows

Run Workflow 
# Run a preset workflow
tapps-agents workflow run workflow-full-sdlc

# Run a custom workflow
tapps-agents workflow run workflows/my-workflow.yaml

# List available workflows
tapps-agents workflow list

Workflow Structure

A workflow YAML file defines steps, agents, and conditions:

Example Workflow 
name: Feature Implementation
description: Complete feature development workflow

steps:
  - name: plan
    agent: planner
    command: plan
    args:
      description: "{{ input.description }}"
    
  - name: design
    agent: architect
    command: design
    depends_on: [plan]
    
  - name: implement
    agent: implementer
    command: implement
    depends_on: [design]
    
  - name: test
    agent: tester
    command: generate
    depends_on: [implement]
    
  - name: review
    agent: reviewer
    command: review
    depends_on: [implement, test]
    
  - name: gate
    agent: orchestrator
    command: gate
    condition:
      review.overall_score >= 70

Available Preset Workflows

  • workflow-full-sdlc - Complete software development lifecycle
  • workflow-quality - Quality assurance and code review
  • workflow-quick-fix - Rapid bug fixes and small changes
  • workflow-rapid-dev - Fast development iteration
  • workflow-maintenance - Code maintenance and refactoring

Project Profiling

Automatic Project Detection

The framework automatically detects project characteristics for context-aware expert guidance:

  • Deployment Type - local, cloud, enterprise
  • Tenancy - single-tenant, multi-tenant
  • User Scale - single-user, small-team, department, enterprise
  • Compliance - GDPR, HIPAA, PCI, SOC2
  • Security Level - basic, standard, high, critical

The profile is automatically generated and stored in .tapps-agents/project-profile.yaml.

Configuration

Project Configuration

The main configuration file is located at .tapps-agents/config.yaml:

Config Example 
project:
  name: "My Project"
  root: "."

mal:
  enabled: true
  ollama_url: "http://localhost:11434"
  default_model: "qwen2.5-coder:7b"
  fallback_provider: "anthropic"
  timeout: 60

scoring:
  enabled: true
  weights:
    complexity: 0.20
    security: 0.30
    maintainability: 0.25
    test_coverage: 0.15
    performance: 0.10
  thresholds:
    overall_score: 70

context:
  tiered_enabled: true
  default_tier: "tier1"
  cache_ttl: 300

Industry Experts

What are Experts?

Industry Experts provide domain-specific knowledge and weighted decision-making. They can be business domain experts (configured per project) or built-in technical experts (16 total).

Built-in Experts

The framework includes 16 built-in technical experts:

  • Security - Security best practices and vulnerability detection
  • Performance - Performance optimization and profiling
  • Testing - Testing strategies and test design
  • Data Privacy - Privacy compliance and data protection
  • Accessibility - Accessibility standards and WCAG compliance
  • UX - User experience design principles
  • Code Quality - Code quality metrics and standards
  • Software Architecture - Architectural patterns and design
  • DevOps - CI/CD and deployment practices
  • Documentation - Documentation standards and best practices
  • AI Frameworks - AI/ML framework guidance
  • Observability - Monitoring and observability
  • API Design - API design patterns and REST principles
  • Cloud Infrastructure - Cloud architecture and services
  • Database - Database design and optimization
  • Agent Learning - Agent learning and improvement

Configuring Industry Experts

Define business domain experts in .tapps-agents/experts.yaml:

Experts Configuration 
experts:
  - id: "finance-expert"
    name: "Finance Domain Expert"
    domain: "finance"
    knowledge_base: "experts/knowledge/finance/"
    weight: 0.51  # Primary expert
    
  - id: "healthcare-expert"
    name: "Healthcare Domain Expert"
    domain: "healthcare"
    knowledge_base: "experts/knowledge/healthcare/"
    weight: 0.24  # Secondary expert

Cursor AI Integration

Cursor Skills

All 13 agents are available as Cursor Skills. After running tapps-agents init, Skills are installed in .claude/skills/.

Use Skills in Cursor by mentioning the agent name:

  • "@reviewer review this file"
  • "@implementer add authentication"
  • "@tester generate tests"

Background Agents

Background Agents offload heavy tasks to cloud/remote agents. Configure in .cursor/background-agents.yaml:

Background Agents Config 
background_agents:
  - name: "Quality Agent"
    agent: "reviewer"
    description: "Code quality review and scoring"
    triggers:
      - "review code"
      - "check quality"
    
  - name: "Testing Agent"
    agent: "tester"
    description: "Generate and run tests"
    triggers:
      - "generate tests"
      - "run tests"

Best Practices

1. Use Appropriate Context Tiers

Let the framework automatically select context tiers based on task complexity. Most tasks use Tier 1 (core context), saving 90%+ tokens.

2. Leverage Workflows for Complex Tasks

For multi-step processes, use workflows instead of running agents individually. Workflows provide better coordination and artifact tracking.

3. Configure Experts for Domain-Specific Projects

Add industry experts for projects with specific domain requirements (finance, healthcare, etc.). Experts provide weighted decision-making.

4. Use Hardware Profiles

Set appropriate hardware profiles (NUC, development, workstation) for optimal performance on your machine.

5. Enable Tiered Context

Keep tiered context enabled to maximize token savings. The system automatically selects the appropriate tier.

6. Review Scores Regularly

Use the Reviewer Agent's scoring system to track code quality over time. Aim for overall scores above 70.

Troubleshooting

Common Issues

Agent Not Found

Ensure the agent name is correct. List available agents:

List Agents 
tapps-agents --help

Model Not Available

Check model availability and configuration:

Check Models 
# For Ollama
ollama list

# Check MAL configuration
tapps-agents doctor

Workflow Fails

Check workflow syntax and dependencies:

Validate Workflow 
# Validate workflow file
tapps-agents workflow validate workflows/my-workflow.yaml
