Coding Practices

Learn how to work effectively with TappsCodingAgents using Vibe coding and multi-file context-aware development. Most development tasks should use workflows rather than single-file edits.

Vibe Coding: The Primary Approach

Vibe Coding: Describe what you want to build at a high level, and let workflows orchestrate multiple agents to create the complete solution across multiple files, with proper architecture and quality checks.

What is Vibe Coding?

Vibe coding is the practice of describing your intent in natural language and letting the AI agents figure out the implementation details across multiple files. Instead of manually editing one file at a time, you describe the feature or change you want, and the workflow handles:

  • Creating/modifying multiple related files
  • Maintaining consistency across the codebase
  • Following architectural patterns
  • Ensuring quality and security
  • Updating related documentation

Vibe Coding Example

Scenario: Add User Authentication

Instead of: Manually creating auth.py, updating routes, adding middleware, writing tests, etc.

Do this:

Vibe Coding Command 
tapps-agents workflow rapid "Add user authentication with JWT tokens, password hashing, and protected routes"

What Happens:

  1. Planner breaks down into stories: registration, login, JWT generation, route protection
  2. Implementer creates/modifies:
    • src/auth/models.py - User model with password hashing
    • src/auth/schemas.py - Pydantic schemas
    • src/auth/routes.py - Registration and login endpoints
    • src/auth/middleware.py - JWT verification middleware
    • src/auth/utils.py - Password hashing utilities
    • Updates src/main.py - Registers auth routes
  3. Reviewer checks security, code quality
  4. Tester generates tests for all auth functionality

Single-File vs Multi-File Development

When to Use Single-File Edits

Single-file edits are appropriate for:

  • Small bug fixes - Fixing a typo, correcting a single function
  • Quick changes - Updating a constant, changing a string
  • Isolated modifications - Changes that don't affect other files
  • Refactoring within a file - Extracting a function, renaming variables
Single-File Edit Example 
# Good for single-file edits
tapps-agents implementer refactor src/utils/helpers.py "Extract the validation logic into a separate function"

When to Use Workflows (Multi-File)

Use workflows (Vibe coding) for:

  • New features - Features that span multiple files
  • API endpoints - Routes, schemas, models, tests
  • Architectural changes - Adding layers, changing patterns
  • Integration work - Connecting multiple components
  • Complete implementations - Full feature with all related files
Multi-File Workflow Example 
# Better for multi-file features
tapps-agents workflow rapid "Add payment processing with Stripe integration, webhook handling, and transaction logging"

Multi-File Context Awareness

TappsCodingAgents uses tiered context to efficiently provide agents with the right amount of context for their tasks.

Context Tiers

Tier 1: Core Context

~500 tokens

  • File structure
  • Function signatures
  • Class definitions
  • Imports/exports

Used by: Analyst, Planner, Reviewer (initial)

Tier 2: Extended Context

~2000 tokens

  • Everything in Tier 1
  • Function bodies
  • Related files
  • Test files

Used by: Architect, Designer, Reviewer (detailed)

Tier 3: Full Context

~10000 tokens

  • Everything in Tier 2
  • Complete file contents
  • Git history
  • Documentation
  • Cross-references

Used by: Implementer, Improver

How Context Works in Workflows

When you run a workflow, each agent receives context based on:

  1. Their default tier (defined in workflow)
  2. Required artifacts from previous stages
  3. Related files automatically discovered
  4. Project profile for domain-specific guidance
Token Efficiency: Tiered context saves 90%+ tokens compared to always loading full files. Most tasks use Tier 1 or Tier 2, only implementation tasks need Tier 3.

Best Practices

1. Use Workflows for Most Tasks

Workflows should be your default choice. They provide:

  • Better coordination between agents
  • Automatic quality gates
  • Artifact tracking
  • Consistent patterns

2. Be Descriptive in Your Prompts

Good prompts include:

  • What you want to build
  • Why it's needed (context)
  • Constraints or requirements
  • Integration points with existing code
Good Prompt Example 
tapps-agents workflow rapid "Add user profile management: users can view and edit their profile, upload avatar images (stored in S3), and update preferences. Must integrate with existing auth system and respect GDPR data export requirements."

3. Review Generated Code

Always review the code generated by workflows:

  • Check that it matches your intent
  • Verify integration with existing code
  • Review security implications
  • Test the functionality

4. Iterate with Workflows

If the first attempt isn't perfect:

  • Use the quality gate feedback
  • Run the workflow again with refinements
  • Use the "fix" workflow for corrections

5. Leverage Project Profile

Ensure your project profile is accurate. It influences:

  • Which experts are consulted
  • Security requirements
  • Compliance checks
  • Architecture decisions

What to Avoid

‚ùå Don't: Edit Files Manually When Workflows Can Help

If your change affects multiple files or requires coordination, use a workflow instead of manual edits.

‚ùå Don't: Use Single-File Edits for Features

Features typically span multiple files. Use workflows to ensure consistency.

‚ùå Don't: Skip Quality Gates

Workflows include quality gates for a reason. Don't bypass them unless absolutely necessary.

‚ùå Don't: Ignore Project Profile

Keep your project profile updated. It guides agent decisions.

‚úÖ Do: Use Workflows as Default

Make workflows your default choice. Only use single-file edits for truly isolated changes.
