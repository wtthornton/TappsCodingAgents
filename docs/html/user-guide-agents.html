Agents Reference

Complete reference for all 13 TappsCodingAgents agents, their commands, use cases, and how to use them individually or in workflows.

Note: While individual agents can be used directly, workflows are the recommended approach for most development tasks. See the Workflows guide for details.

All 13 Agents

Using Individual Agents

While workflows are recommended, you can use individual agents for specific tasks:

Reviewer Agent

Reviewer Commands 
# Review a file
tapps-agents reviewer review src/main.py

# Calculate quality scores
tapps-agents reviewer score src/main.py

# Review with specific model
tapps-agents reviewer review src/main.py --model qwen2.5-coder:7b

Implementer Agent

Implementer Commands 
# Implement a feature
tapps-agents implementer implement "Add user authentication endpoint"

# Refactor code
tapps-agents implementer refactor src/api.py "Extract helper functions"

Tester Agent

Tester Commands 
# Generate tests
tapps-agents tester generate src/main.py

# Run tests
tapps-agents tester run tests/test_main.py

For complete command reference, see the API Reference.

Cursor Integration

All 13 agents are available as Cursor Skills. After running tapps-agents init, Skills are installed in .claude/skills/.

Using Skills in Cursor

Use Skills in Cursor by mentioning the agent name:

  • "@reviewer review this file"
  • "@implementer add authentication"
  • "@tester generate tests"
  • "@architect design the system architecture"

Background Agents

Background Agents offload heavy tasks to cloud/remote agents. Configure in .cursor/background-agents.yaml:

Background Agents Config 
background_agents:
  - name: "Quality Agent"
    agent: "reviewer"
    description: "Code quality review and scoring"
    triggers:
      - "review code"
      - "check quality"
    
  - name: "Testing Agent"
    agent: "tester"
    description: "Generate and run tests"
    triggers:
      - "generate tests"
      - "run tests"
